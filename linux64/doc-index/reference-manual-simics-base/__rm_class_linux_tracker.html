linux_tracker

Provided By
linux-tracker
Description
Linux Tracker
Interfaces Implemented
conf_object, log_object, osa_linux_software_tracker_info, osa_micro_checkpoint, osa_target_info, osa_tracker_control, osa_tracker_parameters

Attributes

check_kernel_version

Optional attribute; read/write access; type: boolean. If true, a check will be made to verify that the parameters or the symbol file matches the kernel in memory

early_detect_check

Optional attribute; read/write access; type: boolean. If true, perform early task struct validation while detecting parameters, this can improve detection time. Having this set to false might improve chance to detect parameters for some targets that would otherwise fail.

enable_vm_support

Optional attribute; read/write access; type: boolean. If true the tracker will ignore all haps and breakpoints that occur while a processor is running a virtualization guest (using KVM). If this is disabled, errors might show up when guests are running. Handling of guests is currently only supported for x86-64 processors. VM support cannot be used together with physical breakpoints.

enabled

Pseudo attribute; read-only access; type: boolean. Tracker enabled - read only

osa_admin

Required attribute; read/write access; type: object. The osa administrator object, must implement the osa_tracker_state_admin interface

parent

Required attribute; read/write access; type: object. The parent object, must implement the osa_machine_query and osa_machine_notification interfaces
