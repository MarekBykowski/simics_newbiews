tcf-agent

Provided By
tcf-agent
Description
This class implements the Simics Debugger API. The class can be instantiated at most one time. Do not create this instance manually. Use SIM_get_debugger instead.
Interfaces Implemented
agent_proxy_finder, conf_object, debug_config, debug_internal, debug_notification, debug_query, debug_setup, debug_step, debug_symbol, debug_symbol_file, frontend_server, log_object, system_panel_frontend, tcf_breakpoints, tcf_channel, tcf_elf, tcf_memorymap, tcf_pathmap, tcf_pe, tcf_serial_tap, tcf_target_identification, tcf_trace

Attributes

panel_state_managers

Session attribute; read-only access; type: [o*]. Connected state managers

parameters

Pseudo attribute; read-only access; type: string. Parameter for the TCF agent.

properties

Pseudo attribute; read-only access; type: dictionary or nil. Properties for the TCF peer.

stepping_mc_interval

Pseudo attribute; read/write access; type: integer. The interval in steps between micro-checkpoints that are planted when doing reverse stepping optimizations. Must be at least 100 steps.

stepping_option

Pseudo attribute; read/write access; type: integer. Option on how stepping over and stepping out is handled:
  • 0 - Breakpoint on function exit address. Longjmp or C++ exceptions might not work correctly. Fastest and default.
  • 1 - Breakpoint on return instructions, handles longjmp and C++ exceptions. Only works on x86-based architectures. Slower.
  • 2 - No breakpoints, handles longjmp and C++ exceptions, but can be really slow for long steps.

system_panels

Session attribute; read/write access; type: [o*]. Connected system panels
