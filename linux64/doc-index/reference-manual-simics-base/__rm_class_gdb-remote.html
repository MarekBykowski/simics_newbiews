gdb-remote

Provided By
gdb-remote
Description
The gdb-remote module allows a GDB session to connect to Simics and control the execution. An object of class gdb-remote is used to accept incoming GDB connection requests.

A GDB binary capable of debugging many Simics target machines is included in the Simics Base package. If you want to build your own gdb, read on.

The following table lists, for each target architectures supported by gdb-remote, the string to give to configure as the --target parameter when building GDB, and any command you may have to enter at the GDB command prompt before connecting to Simics:

Attributes

architecture

Pseudo attribute; read/write access; type: string. Architecture of target.

connected

Pseudo attribute; read-only access; type: boolean. Returns true if the gdb-remote object is connected to a GDB session, false if not.

context_object

Optional attribute; read/write access; type: object or nil. Context object that this GDB session is attached to.

disconnect

Pseudo attribute; write-only access; type: boolean. Disconnects the remote GDB

follow_context

Pseudo attribute; read/write access; type: integer. Set to non-zero if context should be followed.

large_operations

Optional attribute; read/write access; type: integer. Set to non-zero if memory operations received from GDB should be performed as single operations instead of bytewise

listen

Pseudo attribute; read/write access; type: integer. Set to start listening for incoming GDB connections on the specified port. If 0 is specified, an arbitrary available port will be used. Read to get the port currently listened on, or 0 if none.

processor

Pseudo attribute; read/write access; type: object or nil. Processor to connect the GDB stub to.

send_packet

Pseudo attribute; write-only access; type: string. Sends a raw packet from gdb-remote to GDB. The string that this attribute is written with will be sent as a packet to GDB.

send_target_xml

Optional attribute; read/write access; type: boolean. Should an XML target description be sent to GDB, default is true, but can be disabled since it can confuse some clients (e.g. Eclipse on a Linux host).

signal

Pseudo attribute; write-only access; type: integer. Sends a signal to the remote GDB. This makes GDB think the program it is debugging has received a signal. See the signal(7) man page for a list of signal numbers.

use_ipv4

Pseudo attribute; read/write access; type: boolean. Determines if connections should be restricted to IPv4. Default is FALSE
