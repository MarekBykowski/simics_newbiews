telnet_frontend

Provided By
telnet-frontend
Description
Class implementing telnet access to the Simics command line. The telnet_frontend should only be used in a secure environment since the port is open for telnet access from any user.
Interfaces Implemented
conf_object, external_connection_events, log_object, terminal_client
Port Objects
tcp <tcp-server> : telnet-frontend TCP server
unix_socket <unix-socket-server> : telnet-frontend Unix domain socket server

Attributes

interactive

Optional attribute; read/write access; type: boolean. Set to TRUE, default, if future telnet connections will be interactive and FALSE if not. Non-interactive connections are useful for scripted control of the command line since no formatting and color codes are used and the prompt is always synchronous.

max_connections

Optional attribute; read/write access; type: integer. The maximum number of allowed telnet connections. Once this number of connections have been created, no further ones are accepted until the num_connections attribute is reset. If the value is zero, no limit is set.

num_connections

Pseudo attribute; read/write access; type: integer. The number of connections done so far since the object was created or the count was reset by a write to this attribute.

plain_text

Optional attribute; read/write access; type: boolean. Set to TRUE if future telnet connections only should use plain text without any formatting and color codes in the output. Default is FALSE.

port

Pseudo attribute; read/write access; type: integer or nil. TCP listening port for the telnet frontend or NIL if no port should be opened. The port must not be privileged, i.e. the allowed range is [1024, 65535]. A value of 0 means that a random free port will be used.

unix_socket_name

Pseudo attribute; read/write access; type: string or nil. UNIX domain socket for telnet frontend.
