sample_linux_tracker

Provided By
sample-linux-tracker
Description
An example of how to write a tracker for Linux that should work with the qsp-linux-common.simics script.
Interfaces Implemented
conf_object, log_object, osa_micro_checkpoint, osa_tracker_control

Attributes

default_parameters

Pseudo attribute; read-only access; type: dictionary. Default parameters for the QSP platform

enabled

Pseudo attribute; read-only access; type: boolean. Tracker enabled - read only

parent

Required attribute; read/write access; type: object. The parent object. Must implement the osa_tracker_state_admin, osa_machine_query, and osa_machine_notification interfaces
